Fluvius

Real-time geospatial stream processing. Zero JVM. Sub-MB footprint.

Rust Tokio async AGPL-3.0 4 crates 59 tests Linux / macOS / Windows

R-tree Spatial Index

Fast spatial queries over millions of entities in real-time.

  • k-Nearest Neighbor queries
  • Bounding box search
  • Radius queries
  • Thread-safe concurrent updates

Complex Event Processing

Detect temporal and spatial patterns in event streams.

  • Multi-step pattern sequences
  • Spatial constraints (near location X)
  • Configurable time windows
  • Per-entity state tracking

Windowing Engine

Four window types with event-time semantics.

  • Tumbling windows
  • Sliding windows
  • Session windows (gap-based)
  • Count-based windows

Watermarks & Late Events

Event-time processing with out-of-order tolerance.

  • Configurable watermark delay
  • Late event detection
  • Tolerance thresholds
  • Processing-time fallback

Geofence Operator

Real-time geofence entry/exit detection.

  • Multi-zone polygon support
  • Enter/exit/inside/outside events
  • Per-entity state tracking
  • Arbitrary polygon shapes

Proximity Alerts

Detect when entities are near each other.

  • Configurable radius thresholds
  • Haversine distance calculation
  • Multi-entity tracking
  • Real-time alert emission

Trajectory Analysis

Analyze movement patterns of tracked entities.

  • Speed calculation
  • Stop detection
  • Distance accumulation
  • Path smoothing

Spatial Aggregation

Real-time density grids, heatmaps, and clustering.

  • Configurable grid cell size
  • Count / Sum / Mean / Max / Min
  • Unique entity counting
  • Threshold-based emission

Map Matching

Snap GPS points to a road network.

  • Nearest-segment projection
  • Confidence scoring
  • Trajectory matching
  • Road attribute lookup

Checkpointing

Periodic state snapshots for crash recovery.

  • Configurable checkpoint interval
  • Automatic garbage collection
  • State restore on startup
  • JSON-serialized snapshots

Temporal Joins

Correlate events across multiple streams.

  • Inner and left-outer joins
  • Entity + time proximity matching
  • Configurable time windows
  • Automatic buffer expiration

Replay Mode

Replay historical data at configurable speed.

  • Real-time (1x) playback
  • Multiplied speed (10x, 100x)
  • Max-speed (instant replay)
  • Timing-preserving inter-event delays

Prometheus Metrics

Built-in observability in exposition format.

  • Events received/emitted/filtered/late
  • Average processing time
  • Atomic lock-free counters
  • /metrics HTTP endpoint

Kafka Connector

Enterprise event streaming integration.

  • Consumer source (subscribe)
  • Producer sink (publish)
  • Consumer groups
  • rdkafka-ready interface

MQTT Connector

IoT device integration via MQTT protocol.

  • QoS 0/1/2 support
  • Subscriber source
  • Publisher sink
  • Authentication support

Topology DSL (TOML)

Declare pipelines without writing code.

  • Sources: WebSocket, File, Kafka, MQTT
  • Sinks: File, Kafka, MQTT, stdout
  • Operators: filter, geofence, CEP, spatial_agg
  • Checkpoint + replay configuration

Quick Start

# Build from source
git clone https://github.com/GeoLang/fluvius.git
cd fluvius && cargo build --release

# Run with a TOML topology
fluvius run --topology pipeline.toml

# Or use individual commands
fluvius geofence --input events.jsonl --zone "warehouse:10.0,20.0,0.01"
fluvius proximity --input events.jsonl --radius 100.0
fluvius trajectory --input events.jsonl

Example Topology

[pipeline]
name = "fleet-monitoring"

[[pipeline.operators]]
type = "geofence"
name = "depot-zone"
zones = [{ name = "depot", center = [10.0, 20.0], radius = 0.01 }]

[[pipeline.operators]]
type = "spatial_agg"
name = "density"
cell_size_deg = 0.1
function = "count"
threshold = 10

[pipeline.source]
type = "kafka"
brokers = ["localhost:9092"]
topic = "gps-events"
group_id = "fluvius-fleet"

[pipeline.sink]
type = "mqtt"
broker_url = "mqtt://localhost:1883"
topic = "alerts/geofence"

[pipeline.metrics]
enabled = true
port = 9090

[pipeline.checkpoint]
dir = "/var/lib/fluvius/checkpoints"
interval_secs = 30
max_retained = 5
